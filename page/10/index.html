理解 HTTP 幂等性

发表于 2018-02-23 | 分类于 学习笔记
字数统计 3,677 | 阅读时长 14

原文链接

基于HTTP协议的Web API是时下最为流行的一种分布式服务提供方式。无论是在大型互联网应用还是企业级架构中，我们都见到了越来越多的SOA或RESTful的Web API。为什么Web API如此流行呢？我认为很大程度上应归功于简单有效的HTTP协议。HTTP协议是一种分布式的面向资源的网络应用层协议，无论是服务器端提供Web服务，还是客户端消费Web服务都非常简单。再加上浏览器、Javascript、AJAX、JSON以及HTML5等技术和工具的发展，互联网应用架构设计表现出了从传统的PHP、JSP、ASP.NET等服务器端动态网页向Web API + RIA（富互联网应用）过渡的趋势。Web API专注于提供业务服务，RIA专注于用户界面和交互设计，从此两个领域的分工更加明晰。在这种趋势下，Web API设计将成为服务器端程序员的必修课。然而，正如简单的Java语言并不意味着高质量的Java程序，简单的HTTP协议也不意味着高质量的Web API。要想设计出高质量的Web API，还需要深入理解分布式系统及HTTP协议的特性。

使用Geotools导入shp文件到 postgresql

发表于 2018-02-23 | 分类于 学习笔记
字数统计 766 | 阅读时长 4

前言

需要做一个上传shp文件后，直接将 shp 中的数据导入到 postgre 数据库中。

shape文件由ESRI开发，一个ESRI [1]的 shape 文件包括一个主文件（.shp），一个索引文件（.shx），和一个dBASE表（.dbf），其中主文件的后缀就是.shp，投影描述文件（.prj）。它们的文件名必须保持一致。

主文件是一个直接存取，变量记录长度文件，其中每个记录描述一个有它自己的vertices列表的shape。

索引文件中，每个记录包含对应主文件记录离主文件头开始的偏移。

dBASE表包含记录的feature的特征。几何和属性间的一一对应关系是基于记录数目的。在dBASE文件中的属性记录必须和主文件中的记录是相同顺序的。
